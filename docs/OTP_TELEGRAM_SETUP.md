## 🚀 Quick Start

### 1. Configure Telegram Bot

If you don't have a Telegram bot yet:

```bash
# 1. Open Telegram and search for @BotFather
# 2. Send /newbot and follow instructions
# 3. Copy the bot token
# 4. Add to your .env file
```

### 2. Update Environment Variables

Your `.env` file should have:

```env
# Telegram Bot (REQUIRED for OTP)
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_BOT_USERNAME=your_bot_username_here

# JWT (should already exist)
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=7d
JWT_REFRESH_SECRET=your_refresh_secret
JWT_REFRESH_EXPIRES_IN=30d

# HMAC for OTP hashing (already added)
HMAC_SECRET=<auto-generated>

# Google OAuth (should already exist)
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=http://localhost:3001/api/v1/auth/google/callback
```

### 3. Initialize OTP Sheets

The OTP sheets will be automatically created when you:

1. **User logs in with Google** (existing flow)
2. **Backend receives Google auth credentials**
3. **OTP service auto-initializes** and creates the sheets

No manual database setup needed!

### 4. Start the Server

```bash
cd apps/backend
pnpm dev
```

You should see:
```
Server is running on port 3001
⚠️  OTP service not initialized: Google Sheets auth or spreadsheet ID not provided
OTP authentication will be available once user authenticates with Google
```

This is normal! OTP will initialize after first Google login.

## 📊 Google Sheets Structure

After initialization, your spreadsheet will have these new sheets:

| Sheet Name | Purpose |
|------------|---------|
| `otp_users` | User accounts (email/password) |
| `telegram_credentials` | Linked Telegram accounts |
| `otp_requests` | Active OTP codes |
| `recovery_codes` | Account recovery (future) |
| `link_tokens` | Telegram linking tokens (future) |

## 🧪 Testing the Integration

### Test 1: User Registration

```bash
curl -X POST http://localhost:3001/api/v1/otp-auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "username": "Test User",
    "password": "SecurePassword123!"
  }'
```

Expected Response:
```json
{
  "success": true,
  "message": "User registered successfully. Please link your Telegram account...",
  "data": {
    "id": 1,
    "email": "test@example.com",
    "username": "Test User"
  }
}
```

### Test 2: Link Telegram

First, authenticate with Google (existing flow) to get an access token, then:

```bash
# Get your Telegram chat ID by sending /start to your bot
# Then link it:

curl -X POST http://localhost:3001/api/v1/otp-auth/link-telegram \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_google_access_token>" \
  -d '{
    "telegramChatId": "123456789",
    "telegramUsername": "your_telegram_username"
  }'
```

### Test 3: Login with OTP

```bash
# Step 1: Request OTP
curl -X POST http://localhost:3001/api/v1/otp-auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "SecurePassword123!"
  }'

# You'll receive a sessionToken and OTP code on Telegram

# Step 2: Verify OTP
curl -X POST http://localhost:3001/api/v1/otp-auth/verify-otp \
  -H "Content-Type: application/json" \
  -d '{
    "sessionToken": "<session_token_from_step1>",
    "otpCode": "123456"
  }'
```

Expected Final Response:
```json
{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": 1,
      "email": "test@example.com",
      "username": "Test User"
    },
    "accessToken": "eyJhbGc...",
    "refreshToken": "eyJhbGc..."
  }
}
```

## ❓ FAQ

**Q: Do I need PostgreSQL?**  
A: No! The integration uses Google Sheets instead.

**Q: Will this affect my existing Google OAuth login?**  
A: No, both authentication methods work independently.

**Q: Can users use both Google login and OTP login?**  
A: Yes, they're separate authentication flows.

**Q: How do I get my Telegram chat ID?**  
A: Send a message to your bot, then call the Telegram API to get updates.

**Q: Is this secure?**  
A: Yes! Passwords are bcrypt-hashed, OTPs are HMAC-hashed, and expire after 5 minutes.

## 🆘 Troubleshooting

### OTP Service Not Initialized
**Solution:** Make sure to call `initializeOtpService(auth, spreadsheetId)` after Google auth.

### Telegram Bot Not Responding
**Solution:** Check `TELEGRAM_BOT_TOKEN` is correct and bot is active.

### Sheets Not Created
**Solution:** Verify Google Sheets API is enabled and auth has write permissions.

### OTP Not Received
**Solution:** User must start a conversation with the bot first.
